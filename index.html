MUJ Data Recon

Upload LSQ, SF, and DEB files to generate a reconciliation dashboard, MUJ output view, DEB-base mismatch analysis, and export-ready files. This version uses LSQ application number as the base, takes program and enrollment status from SF, and calculates DEB not matching with DEB as the base.

Active SF KPI included Refund initiated KPI included Leading 0 removed before ABC / DEB ID compare Workbook + CSV + JSON exports

Input Files

LSQ file

Base file. CSV or Excel. Needs MUJ Application Number, Admission Status, Nationality, course, ABC / DEB IDs, refund fields.

SF file

Used for application match, program, SF ABC / DEB IDs, and enrollment status. CSV or Excel.

DEB file

Used as the base for DEB not matching. Reads enrollment number and DEB course. CSV or Excel.

Upload LSQ, SF, and DEB files in CSV or Excel format, then click Generate dashboard.
Logic in this build
1. MUJ output base = LSQ application number
2. Program + enrollment status = SF
3. DEB not matching = DEB base
4. ABC / DEB ID compare removes a leading 0 before matching
